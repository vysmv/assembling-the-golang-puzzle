В общем, в книге будет создано небольшое приложение, и на его примере будут рассмотрены основные концепции.

Если сделать беглый обзор, то картина будет такая:

1. **UI**

У нас есть главная страница `https://localhost:4000/`, которая просто выводит список заголовков, дат и ID существующих сниппетов.

Заголовок является ссылкой на конкретный сниппет.  
Также есть возможность зарегистрироваться и залогиниться.

Если пользователь залогинен, то ему открывается возможность создавать сниппеты.  
В общем, идея простая и довольно понятная.

2. **Project**

А теперь давайте бегло посмотрим на устройство приложения.

Оно представляет собой пакет `main` в директории `cmd/web/`.  
Плюс несколько внутренних пакетов:
- `assert`
- `models`
	- `models/mocks` (для тестов)
- `validator`

и пакет `ui`, реализующий встраивание статических файлов в итоговый бинарник.

Сам пакет `main` представляет собой внутреннюю инфраструктуру приложения, и его центром является структура `application`:
```go
type application struct {
	logger         *slog.Logger
	snippets       models.SnippetModelInterface
	users          models.UserModelInterface
	templateCache  map[string]*template.Template
	formDecoder    *form.Decoder
	sessionManager *scs.SessionManager
}
```

Далее, на старте приложения отрабатывает функция `main`.

В `main` собираются данные для структуры `application`:
```go
app := &application{
	logger:         logger,
	snippets:       &models.SnippetModel{DB: db},
	users:          &models.UserModel{DB: db},
	templateCache:  templateCache,
	formDecoder:    formDecoder,
	sessionManager: sessionManager,
}
```

И стартует веб-сервер, которому передается созданный нами маршрутизатор с установленным набором маршрутов и соответствующих хендлеров:
```go
srv := &http.Server{
	Addr:         *addr,
	Handler:      app.routes(),
	ErrorLog:     slog.NewLogLogger(logger.Handler(), slog.LevelError),
	TLSConfig:    tlsConfig,
	IdleTimeout:  time.Minute,
	ReadTimeout:  5 * time.Second,
	WriteTimeout: 10 * time.Second,
}
err = srv.ListenAndServeTLS("./tls/cert.pem", "./tls/key.pem")
```

То есть есть слой кода, отрабатывающий в момент запуска бинарника — он формирует приложение.

И, соответственно, есть слой, который работает по запросу (в нашем случае — по HTTP-запросу).

В общем, по этой книге вы сможете заложить основы построения архитектуры приложения, работы с БД, логирования, работы с шаблонами, middleware, сессиями, формами, тестированием и еще рядом необходимых для старта тем.
