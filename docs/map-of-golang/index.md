## Установка Go

Ну что же...давайте установим Go и посмотрим, что мы получаем в распоряжение.  
Для этого перейдём на [официальный сайт](https://go.dev/dl/) 

Напомню, что все ссылки, примеры команд или кода будут в текстовой версии видео на GitHub. 
Ссылка будет в закрепленном комментарии под видео. 

и скачаем пакет для нашей операционной системы.

<p align="center">
  <img src="img/1.png" alt="скачать Go">
</p>

У меня Linux, поэтому я скачиваю архив для него:

```bash
wget https://go.dev/dl/go1.25.4.linux-amd64.tar.gz
```
И распакую его в директорию `/usr/local`:

```bash
sudo tar -C /usr/local -xzf go1.25.4.linux-amd64.tar.gz
```
А потом добавлю путь к директории с бинарником go в переменную `PATH`.
Для этого добавляем в файл `~/.bashrc`:
директиву `EXPORT`.
```bash
export PATH=$PATH:/usr/local/go/bin
```

Обновим сессию оболочки:
```bash
su $USER
```

И запустим команду:
```bash
go version
```
Отлично! Go установлен.

## Первое знакомство

Теперь мы имеем возможность запускать бинарник `go`.

Тут наблюдается распространённое заблуждение.  
Многие называют этот бинарник компилятором.

Но если быть более точным, то это командная оболочка (интерфейс) ко всему набору инструментов Go-разработчика.

К примеру, реальный компилятор расположен по пути `/usr/local/go/pkg/tool/linux_amd64/compile`.

То есть бинарь `go` — интерфейс к тулчейну Go.

И весь этот набор инструментов не имеет внешних зависимостей, не входящих в поставку Go.

> **Дополнительная информация!**  
> Если не считать зависимостей от ОС (типа системных библиотек или системных вызовов).  
> Но без этого в принципе невозможно запускать бинарник.

В этом проявляется философия Go — встроенные инструменты вместо разрознённых экосистемных зависимостей.

Если, например, посмотреть на мир PHP-разработчика, то там есть:  
- интерпретатор,  
- отдельный менеджер пакетов `composer`,  
- и пакеты для тестирования и анализа кода, устанавливаемые через композер. Которые имеют набор собственных зависимостей. 

То есть PHP-toolchain является разрознённым, без единого центра ответственности и стандартизации.
Я, конечно, не имею в виду, что там хаос и всё плохо — просто в Go эта проблема решена лучше.

Так что такой подход можно смело причислить к достоинствам Go.
